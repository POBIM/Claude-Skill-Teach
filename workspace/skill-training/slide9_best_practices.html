Best Practices สำหรับสร้าง Skill

Code Structure

  • แยก logic ออกจากทำงาน
  • ใช้ functions ที่มี purpose ชัดเจน
  • ทำ modular & reusable
  • Comment code ที่ซับซ้อน

Documentation

  • อธิบาย features ทั้งหมด
  • ให้ usage examples
  • ระบุ requirements
  • อัพเดท README ปกติ

Testing & Validation

  • ทดสอบ edge cases
  • ตรวจสอบ output quality
  • สร้าง validation scripts
  • แจ้ง errors อย่างชัดเจน

Organization

  • output ≠ skill folder
  • ใช้ examples/ folder
  • เก็บ core scripts เท่านั้น
  • ลบ temp files ทั้งหมด
